# Implementation Plan: Track 021 - Environmental Context & Sensory Triggers

## Overview
This track focuses on correlating external sensory triggers (noise, light) with internal physiological stress responses.

---

## 🏗 Sub-tasks

### 1. Ambient Noise Monitoring
-   [ ] Implement a `NoiseMonitorRepository` using the `AudioRecord` or `MediaRecorder` API to measure peak dB levels.
-   [ ] Ensure the implementation is highly battery-efficient (e.g., sampling for 1 second every 60 seconds).
-   [ ] Request necessary `RECORD_AUDIO` permissions.

### 2. Ambient Light Monitoring
-   [ ] Access the onboard light sensor using `SensorManager`.
-   [ ] Record Lux levels during periods of activity or physiological alerts.

### 3. Data Correlation & Tagging
-   [ ] Create a `SensoryContext` data model to store Noise and Light levels alongside a timestamp.
-   [ ] Automatically append `SensoryContext` to any `Alert` generated by the system.
-   [ ] Develop a `CorrelationService` on the phone to analyze which sensory spikes correlate with HR increases.

### 4. Clinician Insights
-   [ ] Add "Sensory Sensitivity" cards to the `HealthDashboard`.
-   [ ] Visualize Noise vs. HR on a combined chart.

---

## ✅ Verification Strategy
-   **Unit Tests:** Test the `CorrelationService` with mock data (e.g., verify it correctly identifies a noise spike preceding an HR spike).
-   **Integration Tests:** Ensure `RECORD_AUDIO` and `SENSOR` permissions are handled correctly.
-   **Manual Testing:** Expose the watch to loud environments (controlled) and verify that the dB level is correctly recorded in the alert log.
